# V7 定时任务配置工作台设计

## 目标

远程配置下位机最多 12 条定时湿度覆盖任务，保留 V7 的“暂存、回读、提交或放弃”事务，不使用运行控制区，也不让任务配置与普通参数表混在一起。

## 页面

配置页新增“定时任务”模块：左侧为任务 1–12 列表，显示启用状态、开始时间和覆盖状态；右侧编辑当前任务的启用、月日时分、持续天数、湿度覆盖开关，以及温湿度 1/2/3 的湿度控制高低阈值。

任务名称不作为下位机控制条件，页面固定显示“定时任务 1–12”。这样避免通过 Modbus 传输任意字符串，同时不影响 `scheduledTasks` 对时间与参数的执行。

## 寄存器方案

采用任务选择器与单任务编辑窗口，避免一次暴露 12 条任务的全部字段并保持地址连续。

| 地址 | 类型 | 内容 |
| --- | --- | --- |
| HR720 | U16 | 当前编辑任务索引，0–11，仅切换查看，不标记暂存修改 |
| HR721 | U16 | 已配置任务数，0–12 |
| HR722 | BOOL | 当前任务启用 |
| HR723–726 | U16 | 开始月、日、时、分；全 0 表示不设开始时间 |
| HR727–728 | U32 | 持续时间，单位天 |
| HR729 | BOOL | 启用湿度覆盖 |
| HR730–735 | F32 × 3 | 温湿度 1/2/3 的湿度控制上限 |
| HR736–741 | F32 × 3 | 温湿度 1/2/3 的湿度控制下限 |

写 HR720 仅切换固件内的编辑任务，不触发配置事务。写 HR721–741 会写入暂存任务；写 HR3 提交后才保存并刷新调度器，写 HR3 放弃则恢复当前任务的已保存内容。

## 校验

- 索引：0–11；任务数量：0–12。
- 启用任务必须有合法月日时分和持续时间。
- 每路湿度下限必须小于上限。
- 配置提交后由现有 `ConfigV6_Validate` 和 `scheduled_tasks_on_config_changed` 进行最终校验和重载。

## RTC 与通信配置

RTC 使用“同步电脑时间”一次性操作，不纳入定时任务或开机持久配置。网络、IEC61850、IEC104 因修改后需要重启对应服务，后续放入独立“高级通信配置”页面，并在提交前给出重启提示。
